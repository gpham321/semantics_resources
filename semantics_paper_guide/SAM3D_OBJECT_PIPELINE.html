SAM3 + SAM3D Object Pipeline

From a single RGB image → text-prompted mask → 3D Gaussian splat (.ply) + UV-textured mesh (.obj) → bidirectional ICP alignment with viser overlay. All deps live on this machine; no HuggingFace Spaces required.

Stages: 1. Launch Gradio UI · 2. Mask via text prompt · 3. Generate 3D (splat + mesh) · 4. Unzip the mesh · 5. Bidir-ICP align + viser overlay · Alternative: pure-CLI path · Reference run (bzh5ej9v, 2026-05-21)

2Get a clean mask via text promptImage → Mask

Open http://localhost:7861 in a browser. Drag an image into the upload panel, type a noun phrase, hit Segment.

What works as a prompt

  • Simple noun phrases: "coffee mug", "protein shaker bottle", "white dish", "black coffee maker"
  • SAM3 has been seen failing on domain-specific names — "protein shake" ↦ 0 detections; "bottle" ↦ 0.98. Always retry with a more generic alternative before lowering confidence.
  • Lower the confidence slider toward 0.2 for novel classes (protein-shake tubs, weird cousin mugs). For everyday objects 0.5 is fine.

What makes a SAM3D-friendly mask

  • Single connected component (the object), no stray holes inside, no background bleed.
  • Tight around the object — SAM3D back-projects the masked RGBA, so loose masks pull background pixels into the 3D model and corrupt the texture.
  • If you need to clean up: download the mask, fix in GIMP / Photoshop / Pillow, re-upload as a custom mask in Stage 3 (the UI also accepts a manually-edited mask).
Tip. Click around in the mask preview panel to confirm the mask is bound to the object you wanted, not a logo / decal on it. SAM3's open-vocab head sometimes prefers texture over geometry.
↓

3Generate 3D — splat + UV-textured meshClick → Download

Click Generate 3D. SAM3D runs with_mesh_postprocess=True, with_texture_baking=True (nvdiffrast on RTX 5090, ~25 s/object). When it finishes, two downloads light up.

DownloadPathContents
PLY ~/Downloads/sam3d_output_<hash>.ply 3DGS Gaussian splat (Inria PLY format) — ~600k Gaussians, ~46 MB. Loadable in load_object_splat.py and gsplat.
Mesh (zip) ~/Downloads/sam3d_output_<hash>_mesh.zip 3-file bundle: .obj (~11k verts), material.mtl, material_0.png (UV-baked texture). ~1–2 MB.
Gotcha. The hash (bzh5ej9v, aav2intn, etc.) is a Python tempfile.mktemp suffix — it's random per click. Both files for one object share the same hash. The mesh zip only exists if you click "Generate 3D" again from the same session, so don't close the browser before downloading both.

5Bidirectional ICP align + viser overlayviser :8087

Trellis (the backbone SAM3D uses) emits the mesh and the splat in different axis conventions — there's a Y↔Z swap baked in. The script tries Rx(0°), Rx(+90°), Rx(-90°), Rx(180°) as init rotations, picks the lowest pre-ICP chamfer, then runs rigid bidirectional ICP (scale fixed at 1). Add --learn-scale to opt into Umeyama scale — measured gain is ~0.3 mm on SAM3D pairs (see ablation).

~/Desktop/armlab_splat_tracking/align_sam3d_mesh_to_splat.py

cd ~/Desktop/armlab_splat_tracking
source ~/Desktop/real2render2real/.venv/bin/activate

python align_sam3d_mesh_to_splat.py \
    --splat ~/Downloads/sam3d_output_${HASH}.ply \
    --mesh  ~/Downloads/sam3d_output_${HASH}/sam3d_output_${HASH}.obj \
    --out   ~/Downloads/sam3d_output_${HASH}/mesh_aligned.obj \
    --port  8087

What the script does

  1. Loads the splat PLY (means + RGB from SH degree-0 + sigmoid'd opacity).
  2. Loads the source mesh verts.
  3. Axis init. Computes pre-ICP chamfer for each of {0°, +90°, −90°, 180°} rotations about X. Keeps the best.
  4. Bidir ICP (rigid by default). Calls run_bidirectional_icp(..., learn_scale=False) from load_object_splat.py — runs forward (mesh → splat means) + reverse (splat means → mesh) correspondences with scale fixed at 1.0. The Gaussian-puffiness floor (~18–20 mm on dense splats) dominates over any sub-1% scale tweak, so rigid is the default. Pass --learn-scale to enable Umeyama scale; useful on objects with substantial mesh-envelope vs Gaussian-cloud asymmetry (e.g. cousin TSDF meshes where scale lands at ~0.16–0.17).
  5. Writes the transformed mesh to --out (verts only; .mtl + texture stay in place — open with any OBJ viewer or Blender).
  6. Launches a viser server with both the splat and the aligned mesh overlaid; transform controls let you nudge manually if needed.
CLI knobDefaultEffect
--port8087viser port — preflight-check that nothing else is listening (ss -tln | grep ':8087') or viser falls back silently
--icp-iters100bidir ICP iters; bump to 200 if chamfer plateaus high
--voxel0.005 mdownsample voxel for KNN correspondences; smaller = more pts = slower
--out(none)where to save the transformed .obj; if omitted the viser still launches but nothing is persisted
--learn-scaleoffopt into Umeyama scale (default rigid). On SAM3D ↔ splat pairs this buys ~0.3 mm; useful only when mesh envelope strongly disagrees with the Gaussian cloud extent

What to look at in viser

  • Toggle the splat / mesh visibility — confirm the textured mesh now sits inside the Gaussian cloud (not 90° rotated, not offset).
  • The fixed transform (Rx init + ICP R + scale) is printed in the terminal. If the script ran via | tail -N in a background bash, stdout is buffered — compute chamfer + scale from the saved mesh_aligned.obj separately (see Reference run).
  • Use the on-screen TransformControls to nudge if needed; the viser GUI has a "Save current transform" button that bakes any manual delta into a sibling .npz.
Why the chamfer floors around 15–25 mm. Gaussian centers sit volumetrically off the surface (puffiness) — a perfect surface mesh will show ~15–25 mm chamfer to splat means no matter what. That's the floor. Scale ratios drop to ~0.7–0.95 depending on object geometry (mesh envelope is tighter than the Gaussian cloud's extent). See reference_splat_mesh_alignment.md for the full discussion.

You still need a SAM3 mask. The same project's CLI:

When to prefer CLI. Batch over a directory, or if you want fine-grained control over the SAM3D config (with_mesh_postprocess, seed, use_vertex_color). The Gradio UI is faster for one-offs because the SAM3 backbone stays warm in memory.

How to reproduce the numbers from the saved .obj

cd ~/Desktop/armlab_splat_tracking
~/Desktop/real2render2real/.venv/bin/python <<'PY'
import numpy as np, plyfile, trimesh, torch
HASH = "bzh5ej9v"
p = plyfile.PlyData.read(f'/home/arm-beast/Downloads/sam3d_output_{HASH}.ply')['vertex']
splat = np.stack([p['x'],p['y'],p['z']], -1).astype(np.float32)
src   = trimesh.load(f'/home/arm-beast/Downloads/sam3d_output_{HASH}/sam3d_output_{HASH}.obj',
                     force='mesh').vertices.astype(np.float32)
algn  = trimesh.load(f'/home/arm-beast/Downloads/sam3d_output_{HASH}/mesh_aligned.obj',
                     force='mesh').vertices.astype(np.float32)

def chamfer(a, b, n=20000):
    rng = np.random.default_rng(0)
    a = a[rng.choice(len(a), min(n,len(a)), replace=False)]
    b = b[rng.choice(len(b), min(n,len(b)), replace=False)]
    at, bt = torch.from_numpy(a).cuda(), torch.from_numpy(b).cuda()
    d = torch.cdist(at, bt)
    return float((d.min(1).values.mean() + d.min(0).values.mean()) / 2)

diag = lambda x: float(np.linalg.norm(x.max(0) - x.min(0)))
print(f'scale     = {diag(algn)/diag(src):.4f}')
print(f'pre-ICP   = {chamfer(src,  splat)*1000:.2f} mm')
print(f'post-ICP  = {chamfer(algn, splat)*1000:.2f} mm')

Troubleshooting

SymptomLikely causeFix
SAM3 returns no mask for any prompt Domain-specific noun; confidence too high Try a generic alternative ("bottle" not "protein shake"; "white dish" not "bowl"); lower confidence to 0.2
SAM3D mesh has background bleed Mask is loose around object Re-mask with higher confidence; or hand-edit the PNG and re-upload
Post-ICP chamfer > 30 mm Best init rotation didn't include the right axis flip; very textured / asymmetric object Re-run with --icp-iters 200; if still high, manually nudge in viser TransformControls and Save
Viser overlay doesn't open / "port 8087 already in use" Prior python process holding the port ss -tlnp | grep :8087 → kill <pid>, then re-launch
"No mesh file generated" in CLI mode nvdiffrast not installed in env; falls back to vertex-color only pip install nvdiffrast inside semantic_splatting env, then re-run
Script blocks; can't read chamfer/scale from stdout Bash | tail -N buffered the output until process exit; viser holds the loop forever Compute chamfer separately from the saved mesh_aligned.obj (snippet in Reference run)
