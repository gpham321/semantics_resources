One Demo, Many Objects

Semantics as a Present (SAP) — semantic 3D Gaussian-splat alignment for scalable robot demonstration synthesis from a single RGB video.

[Author list TBD] · Stanford ARMLab

📥 input: 1 RGB video 📤 output: N×M robot demos 🚫 no depth sensor 🚫 no teleop 🚫 no per-object labels

§· Overall Goal

Make robot demonstrations scale like data: one human video → many robot demos, across many objects, with no depth sensor, no teleoperation, and no per-object labels.

📥 1 RGB video of a human doing the task → 🧩 a library of pre-scanned "cousin" objects → 📤 N × M closed-loop robot demos

What we want to enable

  • Recover an object's per-frame SE(3) trajectory from a single RGB video — no depth, no markers.
  • Graft that motion onto a family of scanned cousin objects via semantic 3DGS alignment.
  • Cover both rigid placement (mug → coffee-machine) and articulated motion (open drawer) in one framework.
  • Replay in simulation to mint an N×M demonstration set that trains a closed-loop policy.

What we deliberately avoid

  • No depth sensor at demo time (BundleSDF / FoundationPose require one).
  • No teleoperation per task (the bottleneck we're attacking).
  • No per-object CAD or annotation (MimicGen / RoboCasa need hand-built models).
  • No retraining per cousin — alignment is closed-form on distilled features.

Why it matters. Imitation learning works when data exists; it does not scale because every demo is hand-collected. If a single phone-shot video can yield dozens of visually diverse, physically valid demos across an object family, the cost of teaching a new manipulation skill drops by orders of magnitude — and the policy inherits the generalization of the cousin set, not the limits of one teleoperator's wrist.

§0 Abstract

Collecting robot demonstrations is the bottleneck in imitation learning. We present SAP, which turns a single RGB video of a human performing a manipulation task into a large, visually diverse set of robot demonstrations — with no depth sensor, no teleoperation, and no per-object annotation. Given reference objects pre-scanned as 3D Gaussian splats, we recover the manipulated object's per-frame SE(3) trajectory directly from the video using distilled-DINO + silhouette + monocular depth-ranking losses. We then graft this motion onto a library of scanned cousin objects via region-weighted, semantically-distilled 3DGS alignment: DINO correspondences anchor functional contact regions, and a closed-form transfer maps both rigid placements and articulated motions onto each cousin. Replaying the grafted trajectories in simulation yields an N×M demonstration set that trains a closed-loop policy.

§1 Introduction

Imitation learning works — when you have demonstrations. Collecting them does not scale: teleoperation is slow and per-object, and existing augmentation either needs hand-built object models (MimicGen, RoboCasa) or a depth sensor at demo time (BundleSDF, FoundationPose).

Our key insight: semantic correspondence beats geometric correspondence for motion transfer. Features distilled from DINO into a Gaussian splat match a drawer-front to a drawer-front, or a mug-contact patch to a mug-contact patch, even across wildly different object bodies. So a motion authored once — by tracking one human video — can be re-targeted onto a whole family of "cousin" objects with zero extra annotation. The pipeline has a crisp contract: one RGB video in, N×M robot demos out.

3.2  RGB-only video → splat pose tracking

We optimize the reference object splat's SE(3) pose per frame (track_with_dino_sam3.py) against four losses: distilled-DINO feature match, RGB photometric, sharpened silhouette, and a DepthAnything-V2 depth-ranking term that supplies metric cues without a depth camera. Two recipe upgrades carry tracking quality on tabletop objects: (i) ground-seeded init — we fit the table plane from the scene splat and seed the multi-start search as an upright orbit about the ground normal rather than a blind WIDE_ROT sphere; and (ii) hard-IoU seed ranking (RANK_BY=equal_iou) so per-seed silhouette IoU dominates the composite score (DINO is only a flip-tiebreaker). Cylindrical objects get a 360° yaw-refinement post-step (SSIM + Sobel) since the base loss is rotation-blind on surfaces of revolution. Tracker masks must be pre-propagated SAM3→SAM2 (--masks-dir); the internal SAM3 silently skips frames at its hard-coded 0.5 confidence.

✅ Frame-0 init. IMG_9232 mug 0.73 / CM 0.90 IoU. With the new ground-seeded + equal_iou recipe, corl_pour mug 0.73 → 0.978.
✅ Per-frame. Silhouette weight 1000 over 120 iters holds a symmetric mug upright across the carry; rotation-refinement post-step removes cylindrical yaw drift on raw → smoothed poses.

3.3  Cousin acquisition & feature distillation

Each cousin is a single photo → 3D splat + UV-textured mesh (SAM3 text mask → SAM3D), or a scanned splat. DINOv3 features are distilled per-Gaussian so downstream alignment is semantic. Text-prompted per-object distillation projects multi-view object probabilities back onto Gaussians.

✅ SAM3D mug: scale 0.944, chamfer 60 → 18.9 mm after bidirectional ICP.
✅ Per-object distill (laptop): 386k → 36.8k Gaussians (9.5%) at p≥0.8, DINO features preserved.

3.4  Region-weighted semantic alignment

A 3-stage ICP grafts each cousin into the reference frame: (i) bidirectional unweighted coarse align, (ii) region-weighted refinement that up-weights DINO-matched contact regions (e.g. the mug-contact patch on the coffee machine), and (iii) a centroid-preserving upright projection. For far-out-of-frame regions (the cm10/11/12 failure mode) the region weight is amplified from 20× → 200× in stage 2 — the threshold recovers the mask, but the weight makes ICP honor it. Part-to-part alignment uses --no-center to keep parent-frame coordinates so one transform moves the cousin's whole scene.

Hub composition for symmetric objects. Cousin → sparse-reference semantic ICP flips ~10% of symmetric mugs (the flipped solution has a marginally lower loss). We instead align cousins → a dense hub splat (no flips), and compose with a one-shot hub → reference transform. Headless align + viewer-side --load-manual-transform sidesteps the one-process GUI hang.

✅ 21 cousins re-aligned to the canonical reference. Scale clusters by class — mugs 0.14–0.18, coffee-machines 0.44–0.58 — and an out-of-cluster scale flags a misclassification automatically.

3.5  Cross-object motion transfer

Given the per-cousin similarity transform (s, R, t), the reference motion grafts in closed form:

# prismatic (drawer) — direction only, translation cancels a_cousin = Rᵀ · a_ref # revolute (screw axis) — direction + on-axis point + scale (ω, p)_cousin = ( R·ω_ref, R·(p_ref·s) + t ) # rigid placement (mug → coffee machine) — full SE(3) compose T_place,cousin = T_align · T_place,ref # scale composes multiplicatively

3.6  Grounding & grasp synthesis

The tracked motion is snapped to a recovered ground plane, and the human grasp is converted to a robot end-effector: HAMER recovers the MANO hand, which is depth-aligned to the splat and mapped to a parallel-jaw xArm TCP via the Phantom/Bohg pinch-to-gripper rule (Z_TCP = pinch − wrist). A "settled-grasp" window picks the rigid-contact frames. RSRD's Z-only HAMER alignment leaves the (X,Y) at HAMER's wrong-z values; we fix this with a DA-V2-Base monodepth pass plus an X,Y re-back-project at the corrected Z — tightening the bias region from 87% → 56% of Gaussians touched.

✅ HAMER align: 72/101 frames, median offset −4.1 cm, splat→meters 0.748. Settled-grasp window frames 64–75 (best ≈ 69, pinch ~28 mm).

One canonical grasp, transferred to all cousins. Rather than re-synthesizing grasps per cousin, we author one force-closure grasp interactively (interactive_hamer_xarm.py, HAMER-biased with a gizmo + perturbation sweep) and transfer it across the cousin family via Sim3, then deepen by +8 mm to absorb residual scale error. The transferred grasps are validated per scale in {0.8, …, 1.2}: 16 mugs × 5 scales = 7202 grasps, 0 dropped, 100% re-validated at the OPEN gripper width (collision is checked at the open width, NOT at contact gap, to avoid a false-positive that the original sampler hit).

Per-episode auto grasp-pose selection. Pre-baked IK tables don't survive cousin pose noise, so the runner picks a grasp at runtime: scan grasps × 6 options (NORMAL/FLIP × home / s-tier / new) and accept the first that clears all of reach, self-collision, and ground. Ground clearance uses the lowest point on each arm-link capsule (link1–7); joint-origin checks miss a forearm dipping 11 cm into the table. Cousin IK tables are also a foot-gun for live re-solves — the runner uses IK branch-lock-replay (pre-compute one warm-started chain from the anchor seed, replay it) plus --no-manipulability, so executed joints stay on the same branch as the shown grasp (verified viser-vs-executed to 0.2°).

✅ Settled grasp #55 transferred to all 16 cousins: 30/30 valid within ≤ 0.5 cm recovery; +4 mm deepen universal; +8 mm breaks two mugs at scale 1.0 only.

3.7  Simulation replay & policy training

Grafted per-cousin trajectories replay in IsaacLab with domain-randomized table material (52-texture pool, MDL-unbind-then-bind workaround for instanced assets), lighting (skybox + sphere lights, color-temperature jitter), camera baseline, and a decomposed L-shaped foam-board poster backdrop mimicking the real RealSense scene. Per-episode goal noise (mug ±6 cm / ±30°, CM ±6 cm / ±12°) and an R2R2R-style CM bias toward the mug (p = 0.7) shape the placement distribution.

Vectorized rendering. The runner is render-latency-bound at ~33 Hz on a single process; we move to a vectorized Xarm7MugTransport ManagerBasedRLEnv that reads the same demo NPZ + grasp tables. Two non-obvious bugs were required to make vec renders match the runner: (i) Fabric tiled-render ignores runtime xformOp:scale on a kinematic prim, so the cousin scale is baked into spawn.scale pre-Fabric (we shard by scale); and (ii) AppLauncher RTX defaults flatten ambient lighting and disable GI/AO, so three carb /rtx settings are forced at launch. With both, RGB L1 drops from 51 → 19.6, background L1 = 5.3, PSNR = 18.6 — the residual is table GI-shadow tone, within DR.

Automated camera alignment. We never hand-tune the eval camera. A SAM3-text mask provides the target silhouette; a persistent --cam-match-watch runner re-renders without restarting IsaacSim; SPSA on a composite of SAM IoU + centroid + area + DINOv3 + RGB (RGB alone games) optimizes 6-DOF camera pose, with a 7th DOF (camera roll) patched in post-set_camera_view. A chain-pass schedule shrinks the step scale 0.08 → 0.001 to escape SPSA basins.

A closed-loop policy (diffusion policy / ACT / π₀ — arch TBD) trains on the synthesized N×M LeRobot v2.1 dataset (state = EE pose(6) + gripper(2) via FK of recorded joints — never the IK target, which freezes during HOME/PREGRASP; gripper command, not observed joint, to avoid PD-jitter flips in the URDF viewer).

✅ IsaacLab data-gen (xArm7, iter 32): Cartesian-approach + grasp-hold continuity → 4/4 success, max track-err 0.060 rad (vs 4.66 rad pre-fix).
✅ Camera align: IoU 0.9561 (front) / 0.9284 (side, 7-DOF, 2026-06-04 capture). 2000-episode shard launched 2026-06-04.
